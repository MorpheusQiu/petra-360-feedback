# Petra品牌中国区 360度反馈系统 — 部署方案分析报告

> 编制日期：2026-06-15 | 版本：v1.0

---

## 一、当前系统技术架构概览

| 层级 | 技术栈 | 关键特征 |
|------|--------|----------|
| **后端** | Flask 3.x (Python)，SQLite 文件数据库 | 单进程开发服务器，19个RESTful API端点，SQLite WAL模式 |
| **前端** | 纯HTML/CSS/JS 单页应用（单文件 ~2700行） | 无前端构建工具链，零npm依赖，响应式CSS（兼容移动端） |
| **认证** | Bearer Token（Header + Cookie兜底） | 用户token=纯数字ID，子管理员token=`sa:`+ID，无JWT/过期机制 |
| **数据存储** | SQLite (`feedback.db` 单文件) | 29人预置用户 + 四维度评价数据表 + 子管理员表 + 系统设置表 |
| **多语言** | 前端I18N字典（zh/en 各~200条） | `data-i18n` DOM属性绑定，语言偏好持久化到DB |
| **权限模型** | 三级：员工 / 子管理员 / 主管理员 | `require_auth` → `require_admin` → `require_main_admin` 装饰器链 |
| **当前运行** | `python app.py` → `localhost:5000` | Flask内置WSGI服务器（仅开发用途，不支持生产并发） |

## 二、方案A：部署到企业微信端

### 2.1 技术路径

企业微信提供两种集成方式：

| 方式 | 说明 | 适用性 |
|------|------|--------|
| **自建应用（推荐）** | 在企业微信管理后台创建"H5自建应用"，提供URL入口 | 适用于已有Web系统，改动最小 |
| **小程序** | 使用企业微信小程序框架开发 | 需重写前端，工作量大，不推荐用于Web系统的迁移 |

**推荐采用"自建应用"模式**：将现有系统部署到公网服务器后，在企业微信后台配置应用主页URL，员工通过企业微信工作台点击进入。

### 2.2 需要改造的部分

| 改造项 | 改造内容 | 工作量评估 |
|--------|----------|-----------|
| **登录认证** | 替换用户名/密码登录→企业微信OAuth2.0授权（静默授权获取userid），用userid匹配users表的en_name字段 | 中等（新增OAuth回调路由，修改login API） |
| **用户同步** | 新增定时任务/手动触发：从企业微信通讯录API同步部门、岗位、汇报关系到users表 | 中等（新增通讯录同步API和后台按钮） |
| **前端适配** | 引入企业微信JSSDK（`jweixin-1.2.0.js`），调用`wx.config`注入权限验证 | 较小（JSSDK引入+初始化，约50行代码） |
| **HTTPS** | 企业微信要求所有应用URL为HTTPS | 必须（Let's Encrypt或采购SSL证书） |
| **域名与备案** | 中国大陆服务器需ICP备案 | 必须（备案周期2-4周，需企业资质） |
| **服务器部署** | Gunicorn + Nginx 替代 Flask dev server | 必须（生产环境基本要求） |
| **消息推送** | 可选：通过企业微信API发送反馈提醒、截止日期通知 | 附加值功能（工作量中等） |

### 2.3 身份映射方案

```
企业微信 userid ──映射──> users.en_name
示例：企业微信 userid "Katrina" → users表 en_name="Katrina"

OAuth流程：
1. 员工点击企业微信工作台应用 → 携带code跳转到后端
2. 后端用code换取userid（调用企业微信API）
3. 用userid查找users表，匹配则生成token，不匹配则拒绝访问
4. 首次登录自动同步通讯录中的部门/岗位信息
```

### 2.4 各维度评估

| 评估维度 | 评分 | 说明 |
|----------|------|------|
| **部署流程复杂度** | ⭐⭐⭐⭐ 较高 | 需：①购买服务器+域名+备案（2-4周）②企业微信应用注册+审核（1-3天）③OAuth对接开发（1-2天）④HTTPS配置⑤生产环境部署 |
| **运维维护成本** | ⭐⭐⭐ 中等 | 企业微信API偶尔更新需适配；JSSDK版本升级；通讯录同步需定期维护 |
| **用户体验** | ⭐⭐⭐⭐⭐ 优秀 | 免密登录（OAuth自动识别），嵌入日常工作流，移动端原生体验，消息推送提醒 |
| **数据安全性** | ⭐⭐⭐ 中等 | 反馈数据流经企业微信服务器；敏感评价数据（尤其是匿名领导力反馈）存放在第三方平台基础设施上；需额外评估合规风险 |
| **与现有系统集成** | ⭐⭐⭐⭐ 良好 | 天然与企业微信通讯录、组织架构融合；可与现有飞书/钉钉等其他企业IM并行使用 |
| **企业内部管理适配性** | ⭐⭐⭐⭐⭐ 优秀 | 中国企业员工日常高频使用企业微信；参与率天然有保障；HR可群发提醒；符合中国职场数字化管理习惯 |

## 三、方案B：部署到独立网页端

### 3.1 技术路径

将现有Flask+SQLite系统部署到云服务器或公司内网服务器，通过浏览器直接访问。

**部署架构**：
```
用户浏览器 ──HTTPS──> Nginx (反向代理) ──> Gunicorn (WSGI) ──> Flask App ──> SQLite
```

| 部署方式 | 说明 |
|----------|------|
| **云服务器（阿里云/腾讯云）** | 标准Web应用部署，需域名+备案 |
| **公司内网服务器** | 仅公司内网/VPN可访问，无需备案 |
| **Docker容器化** | 可选，简化环境配置与迁移 |

### 3.2 需要改造的部分

| 改造项 | 改造内容 | 工作量评估 |
|--------|----------|-----------|
| **生产WSGI服务** | Flask dev server → Gunicorn/uWSGI（支持多worker并发） | 必须（config文件+启动脚本） |
| **反向代理** | Nginx配置：静态文件服务、反向代理、SSL终端 | 必须（nginx.conf配置） |
| **HTTPS** | 公网部署需HTTPS（Let's Encrypt免费证书） | 必须 |
| **用户管理** | 可为HR管理后台增加"批量导入用户"功能（Excel → DB） | 较小（附加值功能） |
| **会话安全** | Token加过期时间、增加JWT机制、CSRF防护 | 建议（安全加固） |
| **数据库** | 如并发量增长，SQLite可迁移至PostgreSQL/MySQL | 可选（当前29人规模SQLite足够） |

### 3.3 各维度评估

| 评估维度 | 评分 | 说明 |
|----------|------|------|
| **部署流程复杂度** | ⭐⭐ 较低 | ①服务器部署Nginx+Gunicorn（半天）②域名解析+SSL（1小时）③无需改动任何业务代码 |
| **运维维护成本** | ⭐⭐ 较低 | 标准Linux服务器运维；依赖少（Python+Nginx）；无第三方平台API依赖 |
| **用户体验** | ⭐⭐⭐ 良好 | 浏览器访问，输入URL→登录→操作；需记忆URL和密码；无推送提醒功能 |
| **数据安全性** | ⭐⭐⭐⭐⭐ 优秀 | 数据完全自主可控；可部署在内网，物理隔离；匿名评价数据不外流；合规性最优 |
| **与现有系统集成** | ⭐⭐ 一般 | 独立系统，不与企业IM/通讯录集成；用户需单独管理账号 |
| **企业内部管理适配性** | ⭐⭐⭐ 良好 | 需额外沟通渠道确保参与率（邮件通知+群消息提醒）；但数据独立性强，适合HR敏感数据管理 |

## 五、推荐方案

### 🏆 推荐：方案B「独立网页端」+ 轻量增强

**核心理由**：

1. **立即可用**：当前系统代码几乎无需改动，部署到服务器即可投入使用。6月15日启动的反馈周期迫在眉睫，方案B可在1-2天内上线。

2. **数据安全优先**：360度反馈涉及匿名领导力评价、上下级互评等高度敏感数据，独立部署确保数据完全自主可控，不存在第三方平台数据泄露风险。

3. **维护简单**：系统每半年使用两次（6月+12月），属于低频应用。投入大量精力做企业微信深度集成的ROI不高。独立部署方案维护成本极低。

4. **轻量增强可弥补体验差距**：
   - **URL便捷访问**：将系统URL配置为企业微信工作台"外部链接"，员工点击即可跳转（无需OAuth对接）
   - **邮件/群通知**：HR通过企业微信群发系统URL+登录指引，确保全员知晓
   - **移动端友好**：当前前端已采用响应式设计（`viewport meta` + 弹性布局），手机浏览器直接可用

### 📋 具体部署步骤（方案B）

```
Step 1: 服务器准备（任选其一）
  ├─ 选项1：公司内网服务器（推荐，数据安全最优）
  │   └─ 仅需内网IP，无需域名备案
  └─ 选项2：阿里云/腾讯云轻量应用服务器
      └─ 需域名+ICP备案（如需公网访问）

Step 2: 环境配置（约30分钟）
  ├─ 安装 Python 3.9+
  ├─ pip install flask flask-cors werkzeug openpyxl gunicorn
  └─ 安装 Nginx

Step 3: Gunicorn配置（gunicorn_config.py）
  ├─ workers=2, bind=127.0.0.1:8000
  └─ systemd service 实现开机自启

Step 4: Nginx配置（nginx.conf）
  ├─ 监听80/443端口
  ├─ 反向代理 → 127.0.0.1:8000
  ├─ 静态文件直接服务（/static/ → 本地目录）
  └─ 可选：Let's Encrypt SSL证书

Step 5: 数据迁移
  └─ 拷贝 feedback.db 到服务器（含预置用户数据）

Step 6: 企业微信辅助
  └─ 在工作台添加"外部链接"→ 指向系统URL
  └─ 群发使用指引（URL + 默认密码 + 截止日期）
```

### 🔮 后续演进路径

如果反馈系统使用频率提升或企业微信深度集成需求增强，可在方案B稳定运行后再考虑方案A迁移：

```
当前（方案B）                   3-6个月后                 6-12个月后
  │                              │                         │
  ├─ 独立Web部署                  ├─ 增加企业微信OAuth登录    ├─ 通讯录自动同步
  ├─ 企业微信添加外部链接          ├─ Token改为JWT            ├─ 消息推送提醒
  └─ 邮件/群通知                  └─ SQLite迁移PostgreSQL    └─ 数据分析仪表盘增强
```

## 六、总结

| 结论项 | 内容 |
|--------|------|
| **首选方案** | 独立网页端部署（方案B） |
| **上线时间** | 1-2天（不含可选备案时间） |
| **核心优势** | 零代码改动、数据安全自控、维护成本极低、立即可用 |
| **体验补偿** | 在企业微信工作台添加外部链接入口 + 响应式移动端适配 |
| **远期规划** | 稳定运行后可逐步增加企业微信OAuth登录等增强功能 |
