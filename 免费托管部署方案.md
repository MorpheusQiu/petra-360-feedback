## 一、现有架构评估

### 1.1 当前技术栈

| 组件 | 技术 | 关键约束 |
|------|------|----------|
| 后端框架 | Flask (Python 3.13) | WSGI 应用，单进程 |
| 数据库 | SQLite (WAL模式) | **需要持久化文件存储** |
| 前端 | 单文件 SPA (index.html) | 纯静态，约1500行，无外部依赖 |
| 认证 | Token-based (Bearer + Cookie) | 自实现，无OAuth |
| 文件导出 | openpyxl (Excel) + csv | 在内存中生成，通过 send_file 返回 |
| 生产服务器 | Waitress (4线程) | WSGI 服务器，channel_timeout=120s |

### 1.2 必须调整的组件

| 组件 | 问题 | 调整方向 |
|------|------|----------|
| **数据库路径** | 硬编码 `DB_PATH = os.path.join(DB_DIR, 'feedback.db')` | 改为环境变量 `DB_PATH` + 确保目录可写 |
| **Secret Key** | 硬编码 `app.secret_key = 'petra-360-...'` | 改为环境变量 `SECRET_KEY` |
| **静态文件** | `send_file('static/index.html')` | 改为 `app.send_static_file()` 或 `render_template()` |
| **端口绑定** | 硬编码 `port=5000` | 改为 `PORT` 环境变量 |
| **SQLite WAL** | 需要文件系统可写权限 | 确保托管平台持久化卷挂载到正确目录 |
| **冷启动** | 免费平台通常会休眠 | 添加 health check 端点 + 保持唤醒机制 |

### 1.3 不需要调整的组件

- 前端 SPA (纯 HTML/CSS/JS，零依赖)
- 所有 API 路由逻辑
- 权限体系 (admin/sub_admin/employee)
- 多语言 I18N 系统
- 数据验证与提交锁定逻辑

## 二、免费平台适配性矩阵

### 2.1 快速对比

| 平台 | Python Flask | 持久化SQLite | 免费额度 | 冷启动 | 部署难度 | 综合评分 |

| **Fly.io** | ✅ | ✅ 3GB卷 | 3台256MB VM | 无休眠 | ⭐⭐ 中 | 8/10 |
| **PythonAnywhere** | ✅ | ✅ 本地文件 | 100s CPU/天 | 无休眠 | ⭐ 低 | 6/10 |

| Cloudflare Pages | ⚠️ Worker only | ⚠️ 需D1 | 10万请求/天 | 冷启动 | ⭐⭐⭐ 高 | 3/10 |

### 2.2 为什么静态托管平台不行

```
GitHub Pages / Netlify / Vercel Static:
  ├── 只能托管 HTML/CSS/JS 文件
  ├── 无法运行 Python 进程
  ├── 无法处理 POST/PUT API 请求
  └── ❌ 完全不可行

Vercel/Netlify Serverless (Functions):
  ├── Python 以 serverless function 运行
  ├── 每次请求启动新实例 → SQLite 文件系统不持久
  ├── 需要引入外部数据库 (PlanetScale/Turso)
  └── ❌ 需要大量重写，成本>收益
```

### 2.3 终极推荐

```
     🏆 主方案: Render
     ├── 最简单: GitHub 推送 → 自动部署
     ├── 免费: 750小时/月 ≥ 24h×30天(不需要满月运行)
     ├── Python 原生 + 持久磁盘
     └── *.onrender.com 免费子域名

     🔄 备选方案: Fly.io
     ├── 持久卷 3GB，无休眠
     ├── 需要安装 flyctl CLI
     └── *.fly.dev 免费子域名

     🆘 应急预案: SSH 反向隧道
```

## 三、主方案：Render 部署（推荐）

### 3.1 平台特性

| 项目 | 免费层限制 | 对本系统的影响 |
|------|-----------|---------------|
| 运行时长 | 750 小时/月 | 仅反馈期2周运行，绰绰有余 |
| 内存 | 512 MB | Flask+SQLite 约50MB，完全足够 |

| 休眠 | 15分钟无请求后休眠 | 用户打开页面即唤醒，30s 冷启动可接受 |
| 构建 | 自动检测 Python + requirements.txt | 零配置 |
| 域名 | `*.onrender.com` | 可嵌入企业微信外部链接 |
| HTTPS | 自动 Let's Encrypt | 免费 SSL 证书 |

### 3.2 部署流程（5步，约15分钟）

#### Step 1: 准备代码 — 创建 `render.yaml`

在项目根目录 `360-feedback/` 下创建：

```yaml
# render.yaml
services:
  - type: web
    name: petra-360-feedback
    env: python
    region: singapore          # 新加坡节点，国内访问较快
    plan: free
    buildCommand: pip install -r requirements.txt
    startCommand: python run_prod.py
    envVars:
      - key: PORT
        value: "10000"
      - key: HOST
        value: "0.0.0.0"
      - key: SECRET_KEY
        generateValue: true     # Render 自动生成安全密钥
      - key: DB_PATH
        value: /var/data/feedback.db
    disk:
      name: sqlite-data
      mountPath: /var/data
      sizeGB: 1
```

#### Step 2: 修改 `run_prod.py` 支持 Render

```python
#!/usr/bin/env python
"""Production Startup — Render / Fly.io / local"""
import sys, os
os.chdir(os.path.dirname(os.path.abspath(__file__)))
sys.path.insert(0, os.path.dirname(__file__))

from waitress import serve
from app import app

HOST = os.environ.get('HOST', '0.0.0.0')
PORT = int(os.environ.get('PORT', 5000))
THREADS = int(os.environ.get('THREADS', 4))

print(f'[Petra 360] Starting on {HOST}:{PORT} (threads={THREADS})')
serve(app, host=HOST, port=PORT, threads=THREADS, channel_timeout=120)
```

#### Step 3: 修改 `app.py` 关键配置

需要改 3 处：

```python
# === 修改1: 数据库路径 (第22-23行) ===
# 原代码:
DB_DIR = os.path.dirname(os.path.abspath(__file__))
DB_PATH = os.path.join(DB_DIR, 'feedback.db')

# 改为:
DB_DIR = os.environ.get('DB_DIR', os.path.dirname(os.path.abspath(__file__)))
DB_PATH = os.environ.get('DB_PATH', os.path.join(DB_DIR, 'feedback.db'))
# 确保目录存在
os.makedirs(os.path.dirname(DB_PATH), exist_ok=True)

# === 修改2: Secret Key (第19行) ===
# 原代码:
app.secret_key = 'petra-360-feedback-2026-h1-secret-key'

# 改为:
app.secret_key = os.environ.get('SECRET_KEY', 'petra-360-fallback-dev-key')

# === 修改3: 添加健康检查端点 (在 # ========== Run ========== 之前) ===
@app.route('/api/health')
def health_check():
    """Render 健康检查 + 防休眠"""
    return jsonify({'status': 'ok', 'time': datetime.now().isoformat()})

@app.route('/')
def index():
    return app.send_static_file('index.html')
```

#### Step 4: 确认 `requirements.txt`

```

werkzeug==3.1.3

```

#### Step 5: 推送到 GitHub + 连接 Render

```bash

# 5. 等待 3-5 分钟 → 获得 URL: https://petra-360-feedback.onrender.com
```

## 四、备选方案：Fly.io 部署

### 4.1 与 Render 的差异

| 对比维度 | Render | Fly.io |
|---------|--------|--------|
| 休眠机制 | 15分钟无请求→休眠 | **无休眠，始终在线** |
| 冷启动 | ~30秒 | **0秒** |
| 部署方式 | GitHub 自动 | `flyctl deploy` CLI |
| 持久存储 | 1GB 磁盘 | 3GB 卷 |
| 配置复杂度 | ⭐ 极低 | ⭐⭐ 中等 |
| 国内访问速度 | 新加坡节点良好 | 可选香港/东京节点 |

### 4.2 部署流程

```bash
# 1. 安装 flyctl
# macOS: brew install flyctl
# Windows: iwr https://fly.io/install.ps1 -useb | iex

# 2. 创建 fly.toml
cat > fly.toml << 'EOF'
app = "petra-360-feedback"
primary_region = "hkg"    # 香港节点，国内访问最优

[build]
  builder = "paketobuildpacks/builder:base"

[env]
  PORT = "8080"
  DB_PATH = "/data/feedback.db"

[mounts]
  source = "sqlite_data"
  destination = "/data"

[[services]]
  internal_port = 8080
  protocol = "tcp"

  [[services.ports]]
    port = 443
    handlers = ["http"]
    force_https = true
EOF

# 3. 创建持久卷
fly volumes create sqlite_data --size 1 --region hkg

# 4. 部署
fly deploy

# 5. 获得 URL
# https://petra-360-feedback.fly.dev
```

### 4.3 Fly.io 注意事项

- 首次部署需要注册账号并绑定信用卡（免费额度不会扣费）
- 香港节点 `hkg` 对国内访问延迟最低（~50ms）
- 持久卷数据不会因重新部署而丢失

## 五、应急预案：本地 SSH 隧道（零平台依赖）

当免费平台不可用时（如 Render 宕机、Fly.io 欠费），可临时使用 SSH 反向隧道：

```bash
# 方案A: localhost.run (无需注册, 即时可用)
ssh -R 80:localhost:5000 nokey@localhost.run
# 输出: https://xxxxx.lhr.life → 即公网URL

# 方案B: serveo.net (备选)
ssh -R 80:localhost:5000 serveo.net
# 输出: https://xxxxx.serveo.net

# 方案C: bore (Rust, 稳定)
bore local 5000 --to bore.pub
# 输出: bore.pub:xxxxx
```

| 特性 | localhost.run | serveo.net | bore |
|------|:---:|:---:|:---:|
| 注册要求 | 无需 | 无需 | 无需 |
| HTTPS | ✅ | ✅ | ❌ (需Nginx) |
| 持久化 | 随SSH断开 | 随SSH断开 | 随进程断开 |
| 稳定性 | ⭐⭐⭐ | ⭐⭐ | ⭐⭐⭐ |

**限制**: 本地机器需保持开机 + 网络稳定。仅适用于临时、紧急场景。

## 六、防休眠策略

免费平台的共性问题是休眠。对每半年用2周的反馈系统来说影响可控，但优化方案如下：

### 6.1 Render 防休眠（可选）

```python
# 前端 index.html 中添加心跳（用户打开页面期间保持唤醒）
// 每10分钟 ping 一次 health API，防止 Render 休眠
setInterval(() => {
  fetch('/api/health').catch(() => {});
}, 10 * 60 * 1000);
```

### 6.2 外部监控（可选）

```bash
# 使用 UptimeRobot (免费) 每5分钟 ping 一次
# URL: https://petra-360-feedback.onrender.com/api/health
# 免费额度: 50个监控，5分钟间隔
```

> **注意**: 反馈期仅2周，休眠的影响有限。用户打开页面时等待30秒冷启动即可正常使用。如果老板在意这30秒，可启用防休眠策略。

## 七、安全加固清单

| 安全项 | 免费平台默认 | 加固措施 |
|--------|-------------|----------|
| HTTPS | ✅ 自动 | 已启用 |
| Secret Key | ❌ 硬编码 | 改为环境变量 `SECRET_KEY` |
| 数据库密码 | N/A (SQLite 无远程访问) | 文件仅限应用进程读写 |
| CORS | ✅ 已配置 | `flask-cors` + token 验证 |
| 密码哈希 | ✅ werkzeug | `generate_password_hash` (pbkdf2:sha256) |
| SQL 注入 | ✅ | 全部使用参数化查询 |
| 暴力破解 | ⚠️ 无 | 建议添加: 5次失败锁定15分钟 |
| 数据备份 | ❌ 需手动 | 建议: 反馈截止后导出 Excel 全量备份 |

### 建议的暴力破解防护（可选添加）

```python
# 在 app.py login 函数开头添加
from collections import defaultdict
from threading import Lock
login_attempts = defaultdict(list)
attempts_lock = Lock()

# 在 login() 函数内:
now = datetime.now()
ip = request.remote_addr
with attempts_lock:
    attempts = [t for t in login_attempts[ip] if (now - t).seconds < 900]
    if len(attempts) >= 5:
        return jsonify({'error': '登录尝试次数过多，请15分钟后重试'}), 429
    login_attempts[ip].append(now)
```
